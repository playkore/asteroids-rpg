# Asteroids RPG: описание игры и механики

Этот документ фиксирует текущую механику игры в том виде, в котором она уже реализована. Его цель — быть базой для обсуждения улучшений, баланса и будущих изменений.

## Коротко о сути

Asteroids RPG — это аркадный экшен про корабль, который летает по секториальному миру, уничтожает астероиды и получает за это опыт. Игра сочетает:

- управляемый корабль с инерцией;
- стрельбу с откатом и небольшой разбросом;
- RPG-прогрессию через уровень, здоровье и урон;
- карту из клеток, где некоторые сектора являются боевыми;
- сохранения в три слота.

## Основной игровой цикл

1. Игрок выбирает новый стартовый seed или продолжает сохранение.
2. Корабль появляется в секторе.
3. Игрок управляет движением и стреляет по астероидам.
4. За уничтожение астероидов начисляется опыт.
5. При наборе опыта повышается уровень, растут максимум HP и урон выстрела.
6. Когда сектор полностью очищен, игрок получает небольшое лечение и бонус XP.
7. При выходе за границы экрана игра перекидывает корабль в соседнюю клетку мира.
8. Состояние сохраняется в активный слот.

## Управление

### Мобильное

- На экране есть виртуальный джойстик для движения.
- Стрельба включается автоматически, пока в секторе есть астероиды.

### Клавиатура

- Стрелки влево/вправо: поворот.
- Стрелка вверх: тяга.
- Клавиша стрельбы: ручной огонь.
- Клавиша restart: перезапуск текущего забега.

## Корабль

У корабля есть:

- позиция;
- скорость;
- угол;
- состояние жив / мёртв;
- временная неуязвимость после урона и при переходе между секторами.

Поведение:

- движение по инерции;
- слабое затухание скорости каждый кадр;
- при управлении джойстиком корабль старается двигаться в направлении ввода;
- при удержании клавиш поворота корабль вращается;
- при тяге вперёд получает ускорение по направлению носа;
- при выстреле получает небольшой recoil.

## Стрельба

Выстрел создаёт пулю, которая:

- стартует из носа корабля;
- летит с фиксированной скоростью;
- живёт ограниченное время;
- получает урон от текущего attack игрока;
- имеет небольшой разброс по углу;
- создаёт muzzle flash, частицы и экранную тряску.

Ограничения стрельбы:

- есть cooldown между выстрелами;
- во время game over стрельба используется для перезапуска;
- автоогонь активен только когда в секторе есть астероиды.

## Прогрессия RPG

### Уровень

Игрок стартует с:

- level 1;
- 0 XP;
- 10 HP;
- 10 max HP;
- 2 attack.

Опыт нужен по формуле:

- `5 + 3 * (level - 1)`.

При повышении уровня:

- max HP увеличивается на 2;
- attack пересчитывается;
- HP полностью восстанавливается;
- остаток XP сохраняется и может сразу дать несколько уровней подряд.

### Награда за астероиды

За уничтожение астероидов начисляется XP:

- большой астероид: 4 XP;
- средний: 2 XP;
- маленький: 1 XP.

### Награда за очистку сектора

Когда боевой сектор полностью очищен:

- игрок получает +2 HP;
- игрок получает +3 XP;
- счётчик пассивной регенерации сбрасывается.

### Пассивная регенерация

Если игрок не в game over и HP не на максимуме, то раз в 10 секунд восстанавливается 1 HP.

## Астероиды

Есть три размера астероидов:

- size 3;
- size 2;
- size 1.

Характеристики зависят от размера и уровня сектора:

- у больших астероидов выше запас здоровья;
- чем выше уровень сектора, тем выше HP и контактный урон;
- при попадании пуля может сдвигать астероид по скорости;
- при уничтожении крупный астероид может раскалываться на дочерние астероиды меньшего размера.

Поведение:

- астероиды движутся внутри экрана;
- при выходе за границы экрана они заворачиваются на другую сторону;
- при столкновении с кораблём наносят контактный урон;
- при попадании показывают вспышку, частицы и краткую визуальную реакцию.

## Мир и сектора

Мир разбит на клетки по координатам `x, y`.

Правила мира:

- по оси X мир зациклен;
- по оси Y можно подниматься вверх и опускаться вниз;
- уровни сектора зависят от `y`, где `y = 0` соответствует уровню 1;
- клетки выше нуля считаются более глубокими и опасными;
- клетки с `y < 0` являются пустотой и не содержат боёв.

### Типы клеток

- `empty` — пустая клетка без астероидов;
- `combat` — боевой сектор.

### Генерация клеток

- клетка определяется seed'ом;
- для боевых клеток в текущей версии стартовый набор — три больших астероида;
- часть клеток становится пустыми;
- состояние клетки хранит, была ли она посещена и очищена, а также сколько астероидов осталось.

### Переходы между клетками

Когда корабль выходит за границу экрана:

- текущий прогресс клетки сохраняется;
- игра переводит корабль в соседнюю клетку;
- корабль появляется на противоположной стороне экрана;
- на короткое время включается неуязвимость;
- соседняя клетка может быть пустой или боевой;
- если клетка уже очищена, астероиды не появляются.

## Смерть и возрождение

Если HP игрока падает до нуля:

- корабль создаёт эффект разрушения;
- игрок возрождается в последней очищенной клетке;
- HP полностью восстанавливается;
- астероиды в текущем боевом состоянии сбрасываются;
- включается длительная неуязвимость на время появления.

Если игрок находится в game over:

- можно перезапустить забег;
- на экране меню доступен старт нового забега или загрузка слота.

## Сохранение

Есть три слота сохранения.

Сохранение фиксирует:

- seed;
- размер экрана;
- текущую и последнюю очищенную клетку;
- состояние корабля;
- параметры игрока;
- прогресс клеток;
- cooldown стрельбы;
- cooldown перехода между клетками;
- накопитель пассивной регенерации;
- счётчик спавна;
- состояние game over.

При загрузке состояние восстанавливается из снапшота слота.

## Интерфейс

HUD показывает:

- HP игрока;
- XP игрока;
- текущий сектор;
- прогресс очистки сектора;
- кнопку меню во время игры.

Стартовый экран показывает:

- seed;
- кнопку Roll для нового seed;
- Continue для последнего слота;
- New Game для создания нового забега;
- Load Game для загрузки существующего сохранения.
